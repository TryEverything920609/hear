NAME

     hear -- macOS speech recognition via the command line

SYNOPSIS

     hear [-hvdms] [-i audio_file] [-l language] [-f audio_fmt]

DESCRIPTION

     hear is a command line interface for the built-in speech recognition
     capabilities in macOS. It supports transcription of both audio files and
     microphone input. If no input is provided, the default behaviour is to
     start listening for audio from the microphone.

     The following flags are supported:

     -s --supported
              Print list of supported languages (locales) for speech recogni-
              tion.

     -l --language lang
              Specify speech recognition language (locale). Default is 'en-
              US'.

     -i --input audio_file
              Input audio file. All formats supported by CoreAudio should work
              (e.g. WAV, MP3, AAC, etc.). If the argument is '-', the audio
              data is read from standard input.

     -f --format format_name
              Specify the format of the audio file (e.g. wav, mp3, aac, etc.)
              Only required when reading audio data from standard input.

     -d --device
              Only use on-device offline speech recognition. The default is to
              use whatever Apple's APIs think is best, which may include send-
              ing data to their servers.  When on-device is not enabled, there
              may be a hard limit to the length of audio that can be recog-
              nized in a single session.

     -m --mode
              Enable single-line output mode (only applies when the input is
              microphone audio).

     -h --help
              Print help and exit.

     -v --version
              Print program name and version, and exit.

     Returns 0 on success and greater than 0 on error or failure.
